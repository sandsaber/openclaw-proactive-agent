# surprise-queue.md — Ideas I Think Would Delight My Human

**Purpose:** a ranked, human-vetted list of things I could build. I
surface the top item periodically; the human picks. I do NOT build
anything here on my own authority — anything involving a Tier 2 step
still files a proposal in `PROPOSALS.md`.

**Write-access:** Tier 0.

---

## Format

```
## <short title>
**Rationale:** <why this would delight — one to two lines>
**Effort:** small (<1h) | medium (half-day) | large (multi-day)
**Side effects:** none | workspace-only | external (Tier 2 proposal required)
**Priority:** 1-10 (10 = most impactful I've seen)
**Evidence:** <what I observed that made me think this would help>
**Status:** idea | proposed | approved | built | dismissed
**Added:** YYYY-MM-DD
```

## Rules

1. Ideas come from observation, not speculation. Each entry cites
   evidence (a human request, a near-miss, a repeated pattern, a
   heartbeat finding).
2. Dismissed ideas stay, marked `Status: dismissed` + one-line reason.
   I don't re-propose.
3. I never start building a `Status: idea`. Status goes `idea` →
   `approved` (via human) → `built`. A `proposed` status means it's in
   `PROPOSALS.md` awaiting approval because it touches Tier 2.
4. At opening checklist, I mention the top-priority idea once. If the
   human ignores, I don't mention again for at least a week.
